# 0.2 Problem Formulation

## The Core Problem: Two Different Prediction Targets

All 5 models predict **BOTH targets**:

| Model | Target | Scale | Meaning |
|-------|--------|-------|---------|
| **Naive** | Both | ~0.01 vol, ~50 VND | Baseline: last value for price, 0 for vol |
| **XGBoost** | Both | ~0.01 vol, ~50 VND | Tree-based gradient boosting |
| **NeuralProphet** | Both | ~0.01 vol, ~50 VND | Deep learning time series |
| **TFT** | Both | ~0.01 vol, ~50 VND | Temporal Fusion Transformer |
| **Hybrid** | Both | ~0.01 vol, ~50 VND | GARCH signal + Ridge regression |

**Key Point:** Compare models WITHIN the same target (price vs price, vol vs vol).

---

## Why Price Prediction is Different from Volatility Prediction

### Example of the Scale Problem

```
XGBoost predicts: Price = 52 VND, Actual = 50 VND → MAE = 2
XGBoost predicts: Volatility = 0.015, Actual = 0.012 → MAE = 0.003

MAE(2) > MAE(0.003)

Does this mean XGBoost is WORSE at price prediction?
→ NO! Different targets have different scales!
```

## Concept 1: Martingale Property (Why Price is Unpredictable)

### Definition
A stochastic process S_t is a **martingale** if:
```
E[S_{t+1} | I_t] = S_t
```

**Translation:** The best predictor of tomorrow's price is today's price.

### Implication for Our Problem
- If stock prices follow martingale, then:
  - E[price_{t+1} | price_t] = price_t
  - Any model trying to predict price will at best match naive
  - Complex models will OVERFIT noise and perform WORSE

### Evidence from Our Results (4-Fold CV)

| Bank | Naive MAE (price) | XGBoost MAE (price) | NP MAE (price) | TFT MAE (price) | Hybrid MAE (price) |
|------|-------------------|---------------------|----------------|-----------------|-------------------|
| BID | 0.8482 | 2.9808 (3.5x worse) | 2.6983 (3.2x worse) | 0.6642 (22% better) | **0.5435 (36% better)** |
| CTG | 0.5846 | 3.5682 (6.1x worse) | 1.6227 (2.8x worse) | 0.4718 (19% better) | **0.3823 (35% better)** |
| VCB | 1.1039 | 4.3386 (3.9x worse) | 3.6241 (3.3x worse) | 0.7951 (28% better) | **0.6466 (41% better)** |

**Conclusion:** Hybrid (GARCH+Ridge) is the ONLY model that beats Naive for price. XGBoost and NP are 200-600% WORSE than Naive. TFT is close to Naive but doesn't beat it. This proves martingale property holds for Vietnam bank stocks.

## Concept 2: Volatility Clustering (Why Volatility IS Predictable)

### Definition
Volatility clustering means:
- High volatility today → likely high volatility tomorrow
- Low volatility today → likely low volatility tomorrow

This is formalized in GARCH(1,1):
```
σ²_t = ω + α·ε²_{t-1} + β·σ²_{t-1}
```

### Why Volatility Persists
- α + β close to 1 → slow decay
- News arrives in clusters (good news followed by good news)
- Market participants react to volatility itself

### Evidence from Our Data

| Bank | α (ARCH) | β (GARCH) | α+β | Persistence |
|------|---------|----------|-----|-------------|
| BID | 0.059 | 0.920 | 0.979 | Very High |
| CTG | 0.090 | 0.888 | 0.978 | Very High |
| VCB | 0.076 | 0.886 | 0.962 | High |

**α+β ≈ 0.97** means volatility decays very slowly → predictable!

### Evidence from Our Results (4-Fold CV)

| Bank | Naive MAE (vol) | XGBoost MAE (vol) | NP MAE (vol) | TFT MAE (vol) | Hybrid MAE (vol) |
|------|-----------------|-------------------|--------------|--------------|------------------|
| BID | 0.0151 | 0.0126 (16% better) | 0.0140 | **0.0103 (32% better)** | 0.0131 |
| CTG | 0.0155 | 0.0111 (28% better) | 0.0142 | **0.0103 (34% better)** | 0.0121 |
| VCB | 0.0114 | 0.0104 (9% better) | 0.0109 | **0.0079 (31% better)** | 0.0117 |

**TFT wins volatility prediction on all 3 banks.** Attention mechanism captures volatility clustering better than tree-based methods. Volatility clustering (α+β≈0.97) makes volatility predictable.

## The Fair Comparison Framework

### Cannot Compare Across Targets
```
Price MAE = 2.0 vs Volatility MAE = 0.003 = UNFAIR
```

### Must Compare Within Target

**For PRICE prediction (5 models):**
- Compare: Naive vs XGBoost vs NP vs TFT vs Hybrid
- Winner: **Hybrid** (only model beating Naive)

**For VOLATILITY prediction (5 models):**
- Compare: Naive vs XGBoost vs NP vs TFT vs Hybrid
- Winner: **TFT** (attention mechanism captures volatility clustering best)

### Conversion for Fair Comparison

All models predict BOTH price and volatility targets. To compare fairly:

**For VOLATILITY target:**
- Compare: Naive(0), XGBoost, NP (converted), TFT (converted), Hybrid
- NP/TFT volatility = |log(predicted_price / previous_price)|

**For PRICE target:**
- Compare: Naive(last price), XGBoost, NP, TFT, Hybrid
- All directly comparable since they all predict price

**Note:** GARCH and Ridge are only used WITHIN Hybrid - not as standalone models.
