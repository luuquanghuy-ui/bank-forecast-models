# 0.3 Fair Comparison Framework

## The 5 Models for Thesis

| Model | Type | Target | Role |
|-------|------|--------|------|
| **Naive** | Baseline | Both | Predict last value / 0 |
| **XGBoost** | Core Model 1 | Both | Tree-based ML |
| **NeuralProphet (NP)** | Core Model 2 | Both | Deep learning time series |
| **TFT** | Core Model 3 | Both | Temporal Fusion Transformer |
| **Hybrid** | Developed | Both | GARCH volatility signal + Ridge |

> **Note:** GARCH and Ridge are ONLY used within Hybrid, NOT as standalone models. They are the REASON why Hybrid was developed.

---

## Why Fair Comparison Matters

**WRONG Approach:**
```
Compare XGBoost MAE (vol=0.010) vs NeuralProphet MAE (price=1.2)
→ Comparing different targets!
```

**CORRECT Approach:**
```
Compare XGBoost MAE vs Naive MAE on VOLATILITY
→ Both predict volatility, same scale

Compare Hybrid MAE vs Naive MAE on PRICE
→ Both predict price, same scale
```

## Evaluation Methodology

### 1. Single Split (70/15/15)
Quick comparison on one test period.

### 2. Four-Fold Walk-Forward
Expanding window for robustness:
```
Fold 1: Train 50% | Val 15% | Test 15%
Fold 2: Train 65% | Val 15% | Test 15%
Fold 3: Train 80% | Val 15% | Test 15%
Fold 4: Train 90% | Val 5%  | Test 5%
```

### 3. Metrics

| Metric | Formula | Use Case |
|--------|---------|----------|
| MAE | mean(|actual - predicted|) | Primary metric |
| RMSE | sqrt(mean((actual - pred)²)) | Penalizes large errors |

### 4. Statistical Tests

#### Diebold-Mariano Test
Tests if one forecast is significantly better than another.

```
H0: Two forecasts have equal accuracy
p < 0.05 → Reject H0 → Significant difference
```

## The Fair Comparison Protocol

### Step 1: Define Target
- Is your question about price or volatility?
- Choose appropriate baseline

### Step 2: Choose Baseline
- Price → Naive (last price)
- Volatility → Naive (predict 0)

### Step 3: Compare Only Same-Target Models
- Never compare price model vs volatility model directly

### Step 4: Convert if Needed
If comparing across targets, use conversion:
```
Price model → Volatility: |log(price_t / price_{t-1})|
```

## Why GARCH and Ridge in Hybrid?

GARCH and Ridge are NOT standalone models. They explain WHY Hybrid was developed:

1. **GARCH** provides volatility signal (volatility has clustering patterns)
2. **Ridge** provides stable regression (doesn't overfit like XGBoost)
3. **Hybrid = GARCH signal + Ridge stability**

This is the thesis contribution: combining statistical (GARCH) with ML (Ridge).

## Key Takeaways

1. **Always compare models on the same target and scale**
2. **Price and volatility have different predictability**
3. **XGBoost wins volatility** (tree-based captures clustering)
4. **Hybrid wins price** (GARCH signal + Ridge stability)
5. **XGBoost overfits on price** (251-510% worse than Naive!)
