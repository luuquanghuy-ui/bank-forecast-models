## Key Findings

### 1. XGBoost Dominates Volatility
- XGBoost consistently outperforms all other models for volatility
- Volatility clustering (ARCH effects) is well-captured by tree-based methods
- Hybrid does NOT improve over XGBoost for volatility

### 2. Hybrid Wins Price Prediction
- Martingale property: E[S_{t+1} | S_t] = S_t
- XGBoost shows severe overfitting (251-510% worse than Naive!)
- Hybrid provides stable ~35-41% improvement over Naive
- GARCH volatility signal + Ridge is the key combination

### 3. Why Hybrid (GARCH + Ridge) Was Developed
- For volatility: XGBoost alone is already optimal
- For price: XGBoost overfits, but GARCH+Ridge hybrid succeeds
- Thesis contribution: combining statistical (GARCH) with ML (Ridge)

### 4. Volatility is Predictable, Price is Not
- Volatility clustering creates exploitable patterns (XGBoost captures these)
- Price follows random walk - only Hybrid can beat Naive

## Why XGBoost Works for Volatility

1. **Tree-based ensemble** captures non-linear volatility clustering
2. **Feature engineering** with lagged returns, RSI, volume improve predictions
3. **Random forest effect** - averaging multiple trees reduces overfitting
4. **Volatility clustering** is a strong pattern that tree models exploit well

## Why Hybrid Works for Price (and XGBoost Doesn't)

1. **Martingale property**: E[S_{t+1} | S_t] = S_t makes raw price unpredictable
2. **XGBoost overfits**: Tree models memorize training data, fail on test
3. **Ridge is stable**: L2 regularization prevents overfitting
4. **GARCH provides signal**: Hybrid uses GARCH volatility as additional feature
5. **Thesis contribution**: combining GARCH signal + Ridge stability
