# 3.1 Root Cause Analysis

## Why Some Models Succeed and Others Fail

---

## Part A: Why GARCH and Hybrid Succeed

### 1. Volatility Clustering is Real

**Evidence from data:**
- α + β ≈ 0.97 for all banks
- This means volatility shocks decay very slowly
- Today's volatility predicts tomorrow's volatility

**GARCH equation:**
```
σ²_t = ω + α·ε²_{t-1} + β·σ²_{t-1}
```

- α captures short-term shock impact
- β captures long-term volatility level
- Together they model the persistence structure

### 2. The Right Target

GARCH and Hybrid predict **volatility**, which IS predictable:
- Volatility clusters
- Volatility mean-reverts
- Past volatility contains information about future volatility

### 3. Theoretical Foundation

GARCH is not a black box:
- Parameters have economic interpretation
- Model is based on decades of financial research
- Results are reproducible and explainable

### 4. Ensemble Synergy

Hybrid combines:
- **GARCH:** Captures volatility structure (econometric)
- **Ridge:** Captures short-term patterns (ML)

Neither is perfect alone, but together they cover more regimes.

---

## Part B: Why NeuralProphet and TFT Fail

### 1. The Martingale Problem

**Stock prices follow:**
```
E[S_{t+1} | I_t] = S_t
```

**This means:**
- The best prediction of tomorrow's price is today's price
- Any learned "pattern" is likely spurious
- Complex models overfit the noise

### 2. The Wrong Target

NP and TFT predict **price**, which is:
- Non-stationary (unit root)
- Essentially random in the short term
- Not predictable beyond naive

### 3. Overfitting to Noise

| Model | Parameters | Training Points | Risk |
|-------|-----------|-----------------|------|
| GARCH | ~4 | 1225 | Low |
| Ridge | ~12 | 1225 | Low |
| NeuralProphet | ~1000 | 1225 | HIGH |
| TFT | ~10000+ | 1225 | VERY HIGH |

**More parameters + less data = severe overfitting**

### 4. Neural Networks Need Signal

Neural networks learn patterns:
- In images: edges, textures, objects (clear patterns)
- In text: grammar, semantics (clear patterns)
- In stock prices: noise disguised as patterns (no real signal)

### 5. Complexity Doesn't Help

**Evidence:**
- V1 (simpler) beats V2 (more complex)
- TFT (most complex) is worst performer
- Ridge (simplest ML) beats RF and Neural nets

**Pattern:**
```
Simplicity → Better performance on this problem
```

---

## Part C: Why Ridge Beats Random Forest

### Ridge vs Random Forest Results

| Bank | Ridge MAE | RF MAE | Winner |
|------|----------:|-------:|--------|
| BID | 0.0086 | 0.0105 | Ridge |
| CTG | 0.0101 | 0.0089 | RF |
| VCB | 0.0068 | 0.0114 | Ridge |

Ridge wins 2/3 banks.

### Why?

1. **Linear patterns dominate**
   - Financial data has weak, linear relationships
   - Non-linear patterns are mostly noise
   - Ridge captures linear relationships well

2. **Less overfitting**
   - RF has many degrees of freedom
   - Each tree can memorize noise
   - Ridge is constrained to linear combinations

3. **Feature selection**
   - Ridge uses all features with weights
   - RF splits on subset of features
   - For this data, simple weighting works better

---

## Part D: Why Hybrid is Better Than Both

### The Ensemble Insight

```
Hybrid = w × GARCH + (1-w) × Ridge
```

**Why ensemble works:**

1. **GARCH:** Captures the slow-moving, persistent component of volatility
2. **Ridge:** Captures deviations from the GARCH prediction
3. **Together:** More robust than either alone

### Adaptive Weight

- Weight is optimized on validation set
- Different banks/periods may favor different weights
- The fact that best weight ≈ 0.45 suggests equal contribution

## Summary: The Complete Picture

| Model | Why it Works/Fails |
|-------|-------------------|
| **GARCH** | Right target (volatility), theoretical foundation |
| **Hybrid** | Combines GARCH + Ridge, adaptive to regimes |
| **Ridge** | Simple, linear, low overfitting |
| **RF** | Too flexible, overfits on small data |
| **NeuralProphet** | Wrong target (price), overfits |
| **TFT** | Wrong target (price), severe overfitting |
| **Naive (price)** | Optimal for martingale process |
| **Naive (vol=0)** | Baseline for volatility |

---

## Key Takeaways

1. **Match model to target:** Volatility is predictable, price is not
2. **Simpler is better:** For this data, simple models beat complex ones
3. **Ensemble helps:** Hybrid combines strengths of GARCH and Ridge
4. **Overfitting is deadly:** With 1225 points, complex models fail
5. **Theoretical foundation matters:** GARCH works because volatility clustering is real
