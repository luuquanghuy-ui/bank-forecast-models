# 1.3 TFT Model (Temporal Fusion Transformer)

## Model Description

**TFT** - Temporal Fusion Transformer (Google/PyTorch Forecasting)

An attention-based deep learning model for time series that combines:
- LSTM encoder-decoder
- Multi-head attention
- Variable selection networks
- Quantile regression

### Configuration Used

| Parameter | V1 Value | V2 Value |
|-----------|----------|----------|
| hidden_size | 16 | 32 |
| attention_heads | 2 | 4 |
| epochs | 15 | 50 |
| learning_rate | 0.001 | 0.01 |
| n_lags | 24 | 24 |

---

## Why TFT Predicts PRICE (Not Volatility)

TFT is designed for multivariate time series forecasting.
- Input: Past prices, volumes, and other features
- Output: Future price prediction

This is fundamentally the same target mismatch as NeuralProphet.

## Why TFT FAILS (Even More Than NeuralProphet)

### Root Cause 1: Extreme Overfitting

| Model | Parameters | vs Naive |
|-------|-----------|----------|
| GARCH | ~4 | Wins |
| Ridge | ~12 | Wins |
| NeuralProphet | ~1000 | Loses 2-3x |
| TFT | ~10000+ | Loses 3-17x |

**More parameters → More overfitting → Worse performance**

### Root Cause 2: Attention Finds Spurious Correlations

Attention mechanisms are designed to find meaningful relationships in data.
- In financial data with low signal-to-noise ratio
- Attention finds patterns that are coincidental
- These don't generalize to test data

### Root Cause 3: Same Martingale Problem

Like NeuralProphet, TFT cannot overcome the fundamental fact that:
```
E[S_{t+1} | I_t] = S_t
```

No amount of attention mechanism changes this.

### Root Cause 4: Complexity Mismatch with Data Size

| Model | Parameters | Training Points |
|-------|-----------|-----------------|
| GARCH | ~4 | 1225 |
| Ridge | ~12 | 1225 |
| TFT | ~10000 | 1225 |

With 10000 parameters and only 1225 training points, TFT has ~8 parameters per data point → severe overfitting.
