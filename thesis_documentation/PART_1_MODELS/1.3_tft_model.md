# 1.3 TFT Model (Temporal Fusion Transformer)

## Model Description

**TFT** - Temporal Fusion Transformer (Google/PyTorch Forecasting)

An attention-based deep learning model for time series that combines:
- LSTM encoder-decoder
- Multi-head attention
- Variable selection networks
- Quantile regression

### Configuration Used

| Parameter | Value |
|-----------|-------|
| hidden_size | 16 |
| attention_heads | 2 |
| epochs | 15 |
| learning_rate | 0.001 |
| max_encoder_length | 24 |
| max_prediction_length | 1 |
| dropout | 0.1 |
| loss | QuantileLoss |

---

## Two TFT Variants

### TFT for PRICE (Original)
- Predicts future price directly
- Suffers from martingale property: E[S_{t+1}|S_t] = S_t
- Severe overfitting due to high parameter count (~14,600 params)

### TFT for VOLATILITY (New - Fixed)
- Predicts volatility (|log_return|) directly as target
- Avoids martingale problem since volatility is predictable
- Uses proper prediction loop with combined dataset approach

## Why TFT FAILS for Price

### Root Cause 1: Extreme Overfitting

| Model | Parameters | vs Naive |
|-------|-----------|----------|
| GARCH | ~4 | Wins |
| Ridge | ~12 | Wins |
| NeuralProphet | ~1000 | Loses 2-3x |
| TFT | ~14600 | Loses 3-17x |

**More parameters → More overfitting → Worse performance**

### Root Cause 2: Martingale Property

Like NeuralProphet, TFT cannot overcome:
```
E[S_{t+1} | I_t] = S_t
```

## Why TFT for Volatility Works Better Than Price

1. **No Martingale Constraint**: Volatility is not a martingale - it has predictable structure via volatility clustering

2. **Lower Complexity Relative to Target**: Volatility is smoother than price, making it easier to model with limited data

3. **Volatility Clustering**: GARCH-style patterns exist in volatility, which attention mechanisms can partially capture

## Bug Fix Documentation

### Original Bug: `predict=True` Mode

The original TFT prediction loop used `TimeSeriesDataSet.from_dataset(training, encoder_data, predict=True)` which only creates predictions for the **last data point**.

### Fix: Combined Dataset with `predict=False`

```python
# Instead of predicting one sample at a time:
# 1. Combine last lookback rows of training with full test set
train_tail = train_d.iloc[-lookback:].copy()
pred_df = pd.concat([train_tail, test_d], ignore_index=True)

# 2. Use predict=False to create encoder-target pairs
pred_dataset = TimeSeriesDataSet.from_dataset(training, pred_df, predict=False)

# 3. Batch predict all at once
raw_preds = tft.predict(pred_dataloader, mode="raw", return_x=True)
# Shape: [batch, decoder_steps, n_quantiles] = [N, 1, 7]

# 4. Extract median quantile (index 4)
all_preds = raw_preds[0].prediction.cpu().numpy()[:, 0, 4]
```
