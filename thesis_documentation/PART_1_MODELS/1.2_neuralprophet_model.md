# 1.2 NeuralProphet Model

## Model Description

**NeuralProphet** - Neural Network extension of Facebook Prophet

A time series forecasting model that combines:
- AR (Autoregression) components
- Neural Network layers
- Seasonality decomposition
- Covariates (features)

### Configuration Used

| Parameter | V1 Value | V2 Value |
|-----------|----------|----------|
| learning_rate | 0.01 | 0.1 |
| epochs | 40 | 100 |
| n_lags | 20 | 20 |
| n_forecasts | 1 | 1 |
| weekly_seasonality | True | True |
| yearly_seasonality | True | True |

---

## Why NeuralProphet Predicts PRICE (Not Volatility)

NeuralProphet is designed for **point prediction** of the target variable.
- Input: Past prices
- Output: Future price prediction

This is fundamentally different from GARCH which predicts variance.

## Why NeuralProphet FAILS

### Root Cause 1: Martingale Property

Stock prices follow a random walk:
```
E[S_{t+1} | I_t] = S_t
```

The best prediction of tomorrow's price is today's price.
- Neural networks try to learn patterns in the noise
- These patterns don't exist
- Result: OVERFITTING → worse than naive

### Root Cause 2: Small Sample Size

| Dataset | Training Points |
|---------|------------------|
| BID | ~1225 |
| CTG | ~1225 |
| VCB | ~1225 |

Neural networks typically need:
- Thousands to millions of points for complex patterns
- 1225 points is insufficient
- Model memorizes noise instead of learning signal

### Root Cause 3: Target Mismatch

| Model | Predicts | Data Type |
|-------|---------|-----------|
| NeuralProphet | Price | Non-stationary |
| GARCH | Volatility | Stationary |

- Prices are non-stationary (trending, unit root)
- Returns are stationary (mean-reverting)
- NeuralProphet was designed for stationary or trend-stationary data

### Root Cause 4: Log-Transform Convexity Bias (V2)

When using log-transform:
```
exp(E[log(S)]) < E[S]
```

The expectation of log is biased downward. This makes V2 predictions systematically too low.

## Converting NeuralProphet to Volatility

To fairly compare NP with GARCH/Hybrid, we convert price predictions to volatility:

```python
NP_volatility = |log(NP_price_pred / NP_price_pred.shift(1))|
```
